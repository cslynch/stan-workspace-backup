## OpenClaw Update Available

**Version:** 2026.3.2 (current: 2026.2.6)  
**Released:** March 3, 2026  

### Relevant Changes

- **Security Hardening** (10+ fixes)
  - SSRF guard for web tools (DNS pinning, proxy bypass opt-in)
  - Webhook request auth-before-body parsing (BlueBubbles, Google Chat)
  - Gateway plaintext ws:// loopback-only default (strict by default)
  - Exec approval hardening (path metachar escaping, cwd revalidation)
  - Credential backup permissions (owner-only 0600)
  - Symlink/hardlink boundary checks (archive extraction, bootstrap context)
  - Prompt spoofing hardening (system-prompt routing, marker neutralization)

- **Gateway & Routing**
  - `openclaw config validate` CLI (new, with --json)
  - TLS subagent pairing (LAN/Docker/non-local paths restored)
  - Control UI route precedence + basePath POST handling
  - Heartbeat hot-reload for models/agent.defaults.model changes
  - Gateway/Node browser proxy profile fallback

- **Telegram**
  - Streaming defaults changed to `partial` (was `off`)
  - DM streaming via `sendMessageDraft` + reasoning/answer preview separation
  - Voice mention gating (optional `disableAudioPreflight`)
  - Models picker compact callback format (fixes 64-byte truncation)
  - Inbound media filename preservation

- **Sessions/Config**
  - Subagent file attachments (base64/utf8 + cleanup)
  - Config env vars applied before implicit provider discovery
  - Session-scoped memory context (sessionId in hooks)

- **Exec Approvals**
  - Regex metachar escaping in path patterns
  - Cwd revalidation + drift denial
  - Wrapper argv semantic preservation

### Recommendation

**UPDATE** — Security critical (SSRF + webhook auth + TLS hardening) + gateway stability (routing, config validation, Telegram streaming) + exec approval hardening are production-relevant.
